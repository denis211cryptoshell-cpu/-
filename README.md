# 🤖 Бот-визитка разработчика

Профессиональный Telegram-бот на **aiogram 3.x** с встроенной CMS для управления контентом и админ-панелью.

## 📋 Возможности

### Для пользователей:
- 👤 Главное меню с кнопками разделов
- 🔒 Обязательная подписка на каналы
- 📱 Адаптивный интерфейс
- ✨ **Чистый чат** — бот редактирует сообщения вместо создания новых

### Для администратора:
- 📝 Редактирование контента всех разделов (включая приветствие)
- 🔘 Управление кнопками меню (названия, видимость)
- 📢 Управление каналами подписки
- 📊 Статистика пользователей и нажатий
- 📨 Рассылка сообщений
- 👥 Поддержка нескольких администраторов
- 🧹 Автоматическое удаление старых бекапов

- **Автоматическое применение** миграций при запуске

- **Основной администратор** (`ADMIN_ID`)
- **Дополнительные администраторы** (`ADMIN_IDS`)
- **Rate limiting не применяется** к администраторам

## 🚀 Быстрый старт

### 1. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 2. Настройка .env

```ini
# Telegram Bot Token (от @BotFather)
BOT_TOKEN=1234567890:ABCdefGHIjklMNOpqrsTUVwxyz

# ID основного администратора (узнать у @userinfobot)
ADMIN_ID=5616949472

# ID дополнительных администраторов (опционально)
ADMIN_IDS=

# Каналы для подписки (опционально)
CHANNEL_IDS=@channel1,@channel2

# Путь к SQLite БД
DATABASE_PATH=data/bot.db

# Уровень логирования (DEBUG, INFO, WARNING, ERROR)
LOG_LEVEL=INFO

# Rate Limiting
RATE_LIMIT_MAX_REQUESTS=10
RATE_LIMIT_WINDOW=60
```

### 3. Настройка каналов (опционально)

1. Добавьте бота в канал как **администратора**
2. Скопируйте ID канала (через @getmyid_bot)
3. Добавьте в `CHANNEL_IDS`

### 4. Запуск

```bash
python main.py
```

### 5. Системная проверка

Перед запуском проверьте систему на ошибки:

```bash
python tests/system_check.py
```

**Проверяет:**
- ✅ Синтаксис Python (все .py файлы)
- ✅ Импорт модулей (31 модуль)
- ✅ Валидация .env
- ✅ Подключение к БД и таблицы
- ✅ Регистрация хендлеров
- ✅ Генерация клавиатур
- ✅ Доступность сервисов

## 🔐 Админ-панель

Доступна по кнопке **🔧 Админка** или команде `/admin`.

### Разделы:

| Раздел | Функции |
|--------|---------|
| 📝 Контент | Редактирование текстов (8 разделов включая приветствие) |
| 🔘 Кнопки меню | Изменение названий, показать/скрыть |
| 📢 Каналы | Добавление/удаление каналов подписки |
| 📊 Статистика | Пользователи, нажатия на кнопки |
| 📨 Рассылка | Отправка сообщений всем пользователям |

### Команды администратора:

| Команда | Описание |
|---------|----------|
| `/admin` | Открыть админ-панель |
| `/cache stats` | Показать статистику кэша |
| `/cache clear` | Очистить весь кэшем |
| `/fix` | Исправить HTML теги в контенте |
| `/fixhtml` | Исправить закрывающие теги `</b>` |

### Редактирование кнопок меню:

1. **Админка** → **🔘 Управление кнопками меню**
2. **✏️ Изменить названия**
3. Выберите кнопку из списка
4. Отправьте новое название
5. ✅ Готово!

## 🗃️ PostgreSQL (Supabase)

Для миграции на облачную PostgreSQL:

1. Создайте базу в [Supabase](https://supabase.com)
2. Скопируйте Connection String
3. Добавьте в `.env`:
   ```ini
   DATABASE_URL=postgresql://postgres:password@db.xxx.supabase.co:5432/postgres
   ```
4. Запустите миграцию данных:
   ```bash
   python migrate_sqlite_to_postgres.py
   ```
5. Перезапустите бота

## 📝 Примечания

1. Бот должен быть **администратором** в каналах для проверки подписки
2. При первом запуске создаётся БД с дефолтным контентом
3. Логи сохраняются в `logs/`
4. **Автобекап:** ежедневно в 00:00 в `data/backups/`
5. **Rate limiting:** 10 запросов в 60 секунд (не для админов)
6. **Кэш:** сбрасывается при перезапуске бота
