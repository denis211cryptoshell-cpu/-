# 🤖 Бот-визитка с админ-панелью

Профессиональный Telegram-бот на aiogram 3.x с встроенной CMS для управления контентом.

## 📋 Возможности

### Для пользователей:
- 👤 Главное меню с кнопками разделов
- 🔒 Обязательная подписка на каналы
- 📱 Адаптивный интерфейс
- ✨ **Чистый чат** — кнопки не засоряют историю

### Для администратора:
- 📝 Редактирование контента всех разделов
- 🔘 Управление кнопками меню (названия, видимость)
- 📢 Управление каналами подписки
- 📊 Статистика пользователей и нажатий
- 📨 Рассылка сообщений
- 👥 Поддержка нескольких администраторов

## 🆕 Нововведения

### ✨ Редактирование сообщений (v3.0)
- **Автоматическое редактирование** последнего сообщения вместо создания нового
- **Сохранение message_id** в БД для каждого пользователя
- **Умное удаление** сообщений кнопок (только для не-админов)
- **Чистый чат** — одно сообщение с контентом вместо десятка

**Как работает:**
```
1. Пользователь нажимает "👤 Обо мне"
2. Сообщение кнопки удаляется (для не-админов)
3. Бот редактирует своё последнее сообщение с новым контентом
4. В чате чисто — нет горы сообщений!
```

**Преимущества:**
- 🧹 **Чистая история чата** — нет мусора от кнопок
- ⚡ **Быстрая навигация** — не нужно скроллить
- 🎯 **Удобный UX** — контент обновляется в одном сообщении

### 🔧 Система миграций БД (v3.0)
- **Автоматическое применение** миграций при запуске
- **Безопасное обновление** — все данные сохраняются
- **Поддержка SQLite и PostgreSQL**
- **Версионирование схемы** БД

**Миграции:**
- `001_add_last_message_id` — добавлена колонка `last_message_id` для редактирования сообщений

**Что делает миграция:**
```sql
ALTER TABLE users ADD COLUMN last_message_id INTEGER DEFAULT NULL
```
- ✅ **Не удаляет данные** — только добавляет колонку
- ✅ **Безопасна для продакшена** — существующие записи не меняются
- ✅ **Обратная совместимость** — бот работает и без миграции

### 📬 Универсальная обработка ссылок (v3.0)
- **Все форматы ссылок** поддерживаются:
  - `@username`
  - `username` (без @)
  - `t.me/username`
  - `https://t.me/username`
  - `t.me/+invitehash` (private link)
  - `-1001234567890` (ID)
- **Автоматическое определение** типа ссылки
- **Преобразование в единый формат**

### 🚀 Кэширование (v2.0)
- **In-Memory кэш** с TTL для снижения нагрузки на API Telegram
- **Автоматическая очистка** просроченных записей каждые 5 минут
- **Декоратор `@cached()`** для простого добавления кэширования
- **Команда `/cache`** для управления кэшем (статистика, очистка)

**Что кэшируется:**
- Проверка подписки на каналы (TTL: 60 сек)
- Контент разделов (TTL: 300 сек)

**Преимущества:**
- ⚡ Быстрые ответы бота (1-5 мс вместо 200-500 мс)
- 🛡️ Защита от rate limit Telegram (меньше запросов к API)
- 📉 Снижение риска блокировки бота

### 👥 Мульти-админ (v2.0)
- **Основной администратор** (`ADMIN_ID`)
- **Дополнительные администраторы** (`ADMIN_IDS`)
- **Единая проверка прав** через `settings.is_admin(user_id)`
- **Rate limiting не применяется** к администраторам

## 🚀 Быстрый старт

### 1. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 2. Настройка .env

Откройте файл `.env` и заполните:

```ini
BOT_TOKEN=1234567890:ABCdefGHIjklMNOpqrsTUVwxyz  # от @BotFather
ADMIN_ID=123456789  # ID основного администратора (узнать у @userinfobot)
ADMIN_IDS=987654321,112233445  # ID дополнительных администраторов (опционально)
CHANNEL_IDS=@channel1,@channel2  # каналы для подписки (опционально)
```

### 3. Настройка каналов (опционально)

Если используете обязательную подписку:
1. Добавьте бота в канал как **администратора**
2. Скопируйте ID канала (через @getmyid_bot)
3. Добавьте в `CHANNEL_IDS` в `.env`

### 4. Запуск

```bash
python main.py
```

## 🔐 Админ-панель

Доступна по кнопке **🔧 Админка** в главном меню или команде `/admin`.

### Разделы:

| Раздел | Функции |
|--------|---------|
| 📝 Контент | Редактирование текстов разделов |
| 🔘 Кнопки меню | Изменение названий, показать/скрыть |
| 📢 Каналы | Добавление/удаление каналов |
| 📊 Статистика | Пользователи, нажатия на кнопки |
| 📨 Рассылка | Отправка сообщений всем |
| 🗑️ Кэш | Управление кэшем (`/cache stats`, `/cache clear`) |

### Команды администратора:

| Команда | Описание |
|---------|----------|
| `/admin` | Открыть админ-панель |
| `/cache stats` | Показать статистику кэша |
| `/cache clear` | Очистить весь кэш |
| `/fix` | Исправить HTML теги в контенте |
| `/fixhtml` | Исправить закрывающие теги `</b>` |

## 🔧 Обновление с предыдущих версий

### С v2.0 до v3.0

**Миграция БД применяется автоматически** при первом запуске v3.0!

```bash
# 1. Сделайте бекап БД (на всякий случай)
cp data/bot.db data/bot.backup.db

# 2. Обновите файлы проекта
git pull origin main

# 3. Установите зависимости (если изменились)
pip install -r requirements.txt

# 4. Запустите бота — миграция применится автоматически
python main.py
```

**Что изменится:**
- ✅ Все данные сохранятся
- ✅ Добавится колонка `last_message_id` в таблицу `users`
- ✅ Начнёт работать редактирование сообщений
- ✅ Кнопки начнут удаляться (для не-админов)

**Откат к v2.0 (если нужно):**
```bash
# Удалить добавленную колонку (SQLite ≥ 3.35.0)
sqlite3 data/bot.db "ALTER TABLE users DROP COLUMN last_message_id;"

# Или просто удалите БД (данные будут потеряны!)
rm data/bot.db
```

## 🛠 Технологии

- **Python 3.10+**
- **aiogram 3.x** — асинхронный фреймворк для Telegram Bot API
- **aiosqlite** — асинхронная работа с SQLite
- **pydantic-settings** — валидация настроек
- **loguru** — удобное логирование
- **APScheduler** — планировщик задач (автобекап, очистка кэша)

## 📝 Примечания

1. Бот должен быть **администратором** в каналах для проверки подписки
2. При первом запуске создаётся БД с дефолтным контентом
3. Логи ошибок сохраняются в `logs/errors.log`
4. **Кэширование:** данные кэшируются в памяти (сброс при перезапуске)
5. **Мульти-админ:** все администраторы имеют полные права
6. **Автобекап:** ежедневно в 00:00 создаётся копия БД в `data/backups/`
7. **Редактирование сообщений:** работает только для сообщений бота
8. **Удаление кнопок:** только для не-админов (админам нужно для редактирования)
9. **Миграции БД:** применяются автоматически, безопасны для данных
