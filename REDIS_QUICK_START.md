- Локальный кэш (In-Memory) работает по умолчанию
- Redis полностью совместим и готов к использованию

---

## ⚡ Быстрое переключение

### 1. Сейчас используется (по умолчанию):

```bash
CACHE_BACKEND=local
```

**Преимущества:**
- ✅ Не требует Redis
- ✅ Быстро
- ✅ Работает из коробки

**Недостатки:**
- ❌ Кэш теряется при перезапуске бота
- ❌ Не работает при нескольких инстансах бота

---

### 2. Для включения Redis:

**Шаг 1:** Убедитесь, что Redis запущен
```bash
redis-cli ping  # Должен вернуть PONG
```

**Шаг 2:** Измените `.env`:
```bash
CACHE_BACKEND=redis
REDIS_URL=redis://localhost:6379/0
```

**Шаг 3:** Перезапустите бота
```bash
# Остановите бота (Ctrl+C)
python main.py
```

---

## 🔍 Проверка

### Узнать текущий бэкенд:
```bash
/cache stats
```

В ответе будет указан тип бэкенда:
- `LocalCacheBackend` — In-Memory
- `RedisCacheBackend` — Redis

---

## 🛡️ Безопасность

### Если Redis в сети (не localhost):

```bash
# С паролем
REDIS_URL=redis://:your_password@redis.example.com:6379/0
```

### Рекомендации:
1. Не открывайте Redis в интернет без пароля
2. Используйте файрвол для ограничения доступа
3. Рассмотрите Unix socket для локального подключения

---

## 📊 Сравнение бэкендов

| Параметр | Local | Redis |
|----------|-------|-------|
| Скорость | ⚡ Очень быстро | ⚡ Быстро (сеть) |
| Персистентность | ❌ Нет | ✅ Да |
| Масштабирование | ❌ 1 инстанс | ✅ Несколько инстансов |
| Требует Redis | ❌ Нет | ✅ Да |
